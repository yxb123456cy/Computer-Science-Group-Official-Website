# 高校计算机科技小组官网 - 详细项目需求及界面规划

# 一、项目概述

本项目为高校计算机科技小组定制开发静态官网，基于 Vue3 + TypeScript + Vite + ArcoDesign Vue 技术栈构建，部署于 Vercel 与 Render 双平台。官网核心定位是“展示社团风采、沉淀技术成果、吸引新生加入、提供资源服务”，面向在校学生、计算机爱好者、校企合作方等用户群体，通过简洁直观的界面、清晰的信息架构，实现社团信息的高效传递与用户的便捷交互。

# 二、详细项目需求

## （一）核心目标

1. **展示性目标**：全面呈现社团的组织架构、活动动态、技术成果、荣誉资质，树立专业、有活力的科技社团形象。

2. **转化性目标**：为新生及计算机爱好者提供清晰的招新入口与报名渠道，降低加入门槛，提升招新效率。

3. **实用性目标**：沉淀社团技术资料，整合优质学习、竞赛、开发资源，成为社团成员及校内计算机爱好者的实用工具站。

4. **便捷性目标**：采用静态架构，降低开发与维护成本，支持双平台一键部署，确保快速上线与稳定运行。

5. **兼容性目标**：适配桌面端、平板、移动端等多设备，保证不同场景下的良好浏览体验。

## （二）技术需求

1. **技术栈规范**：采用 Vue3 作为前端框架，使用 Composition API 编写组件；引入 TypeScript 实现类型校验，保证代码健壮性；使用 Vite 作为构建工具，要求热更新响应时间 ≤ 300ms，生产环境打包体积精简，首屏加载时间 ≤ 3s。

2. **UI 组件库要求**：基于 ArcoDesign Vue 组件库开发，复用其布局、导航、展示、交互类组件，减少自定义开发成本；支持主题定制，可修改为社团代表色（如科技蓝、活力橙等），保证界面风格统一且贴合社团调性。

3. **样式需求**：采用 CSS3 + SCSS 编写样式，实现样式模块化管理；支持响应式布局，通过媒体查询适配不同屏幕尺寸；页面视觉层次分明，字体、间距、色彩搭配符合高校学生审美，避免过度复杂设计。

4. **部署需求**：支持 Vercel 与 Render 双平台部署，可通过关联 GitHub 仓库实现自动构建与部署；部署后生成的在线链接稳定可用，支持 HTTPS 协议；静态资源（图片、样式、脚本）加载路径配置正确，无404错误。

5. **性能优化需求**：图片资源需压缩处理（建议使用 TinyPNG 等工具），大型图片可采用懒加载；移除冗余代码与依赖，通过 Tree Shaking 精简打包体积；避免不必要的 DOM 操作，提升页面交互流畅度。

6. **可维护性需求**：代码结构清晰，遵循 ES6+ 规范，添加必要的注释；静态数据与业务逻辑分离，方便社团成员后续修改内容（如活动信息、招新要求等）；项目文档完整，包含 README.md 及关键模块说明。

## （三）功能需求

### 1. 基础导航功能

- 顶部导航栏：包含“首页、关于我们、核心活动、项目成果、成员风采、加入我们、资源导航、联系我们”8个核心入口；桌面端完整展示，移动端自动折叠为汉堡菜单，点击展开导航列表。

- 面包屑导航：在“关于我们、核心活动、项目成果”等二级页面顶部显示，标识当前页面位置，支持点击回溯上级页面。

- 回到顶部按钮：页面滚动距离超过 300px 时自动显示，点击后平滑滚动至页面顶部；按钮样式与页面风格统一，hover 时有交互反馈。

### 2. 首页功能

- Hero 轮播区域：展示社团醒目标语（如“以技术赋能兴趣，以实战提升能力”）、3-5张核心视觉图（活动实拍、项目成果截图、社团合影）；轮播图支持自动切换（间隔 5s）、手动切换，切换时有淡入淡出过渡效果。

- 核心亮点预览：以卡片形式展示 3-4 个社团核心优势（如“每周技术分享、全国竞赛获奖、校企合作资源、实战项目历练”），卡片 hover 时有上浮或阴影加深效果。

- 快速入口模块：设置“查看活动”“了解招新”“浏览成果”3个按钮，分别跳转至对应核心页面；按钮样式突出，位置显眼。

- 近期动态模块：展示 2-3 条最新社团活动或公告（如“2024秋季招新启动”“编程打卡活动圆满结束”），包含标题、时间、简要描述，支持点击查看详情。

### 3. 关于我们页面功能

- 社团简介：详细说明社团成立时间、宗旨、核心定位、服务范围，文本简洁易懂，段落清晰。

- 组织架构：以树形或卡片布局展示核心部门（技术部、策划部、宣传部、外联部），标注各部门职责及负责人信息（可选配头像）。

- 发展历程：以时间线形式展示社团关键节点（如“2022年成立”“2023年获省级竞赛一等奖”“2024年新增AI兴趣小组”），时间线节点包含年份、事件描述、配图（可选）。

- 荣誉资质：以网格卡片形式展示竞赛获奖证书、校企合作授权、优秀项目表彰等图片；支持点击图片预览放大，预览弹窗支持关闭、左右切换。

### 4. 核心活动页面功能

- 活动分类标签：顶部设置“常规活动、特色活动、活动预告”3个标签，点击切换对应内容；标签有选中状态标识（如底部下划线、颜色变化）。

- 往期活动回顾：按时间倒序展示往期活动，每个活动以卡片形式呈现，包含活动标题、时间、地点、3-4张实拍图、简要总结；支持点击卡片查看完整活动回顾（可选弹窗或新页面）。

- 近期活动预告：展示即将开展的活动，包含活动名称、时间、地点、报名方式（如QQ群报名、表单报名）；可设置倒计时组件（针对重要活动）。

- 活动筛选：可选添加时间筛选（如“近3个月”“本学期”“上学期”），快速筛选所需活动。

### 5. 项目成果页面功能

- 项目分类筛选：顶部设置“社团自研项目、竞赛获奖项目、成员优秀作品”分类选项，支持下拉筛选或标签切换。

- 项目卡片展示：以网格布局展示项目，每个卡片包含项目名称、技术栈标签（如“Vue3+Vite”“Python+Django”）、核心功能简介、2-3张演示截图；卡片hover时有交互效果，点击可查看项目详情。

- 项目详情模块：可选以弹窗或新页面形式展示，包含项目详细介绍、开发背景、核心功能演示、技术难点、在线演示链接（如有）、GitHub仓库链接（如有）。

### 6. 成员风采页面功能

- 核心团队展示：以网格卡片形式展示社长、副社长、各部门负责人，每张卡片包含头像、姓名、年级专业、职务、技术方向、个人简介（1-2句话）。

- 优秀成员展示：展示往期优秀成员（如毕业学长学姐入职大厂、竞赛获奖选手），包含个人照片、姓名、成就简介（如“2024年蓝桥杯省赛一等奖”“入职字节跳动前端开发”）。

- 成员寄语：可选添加1-3条社团成员加入感悟（如“在这里找到志同道合的伙伴，技术提升飞快”），增强页面亲切感。

### 7. 加入我们页面功能

- 招新要求：分部门说明招新条件，强调包容性（如“无编程基础可加入，只要有学习热情”）；技术部可标注“具备基础编程能力优先”等针对性要求。

- 报名方式：清晰展示招新QQ群二维码（支持长按识别）、微信公众号二维码、报名表链接（如腾讯文档/问卷星）；二维码下方标注对应说明（如“扫码加入招新群，获取最新信息”）。

- 招新时间：明确集中招新周期（如“每年9月开学后两周”），说明常年招新政策（如“全年可申请加入，联系对应部门负责人”）。

- FAQ模块：以折叠面板形式展示新生高频问题（如“没有编程基础能加入吗？”“社团活动时间是什么时候？”“加入后能获得什么成长？”），点击问题展开答案，默认收起。

### 8. 资源导航页面功能

- 资源分类标签：顶部设置“学习资源、开发工具、竞赛资源、社团仓库”4个分类标签，点击切换对应资源列表。

- 资源列表展示：每个资源包含名称、简要说明（如“LeetCode - 算法刷题必备平台”）、跳转链接；链接支持新窗口打开，避免离开官网。

- 社团仓库：展示社团GitHub/Gitee公共仓库链接，说明仓库用途（如“存放项目代码、学习资料、活动总结”）。

- 资源备注：对关键资源添加星级标识（如★★★推荐），方便用户快速识别优质资源。

### 9. 联系我们页面功能

- 官方账号汇总：以网格布局展示QQ群、微信公众号、GitHub、微博（如有）等官方账号的二维码及对应标识，二维码清晰可识别。

- 联系方式列表：展示社团官方邮箱（如xxjsjst@163.com）、负责人联系方式（可选，建议留QQ/微信）；文本样式可复制，方便用户保存。

- 反馈表单：嵌入第三方表单工具（如问卷星、腾讯文档），设置“问题类型”“反馈内容”“联系方式”3个核心字段，方便用户提交建议或咨询。

- 地址信息：标注社团办公室地址（如有）、常规活动场地（如“计算机楼B203教室”），可选配简易地图截图（静态图片，无需交互）。

### 10. 404页面功能

- 错误提示：展示“404 - 页面不存在”清晰提示文本，说明可能原因（如“链接错误、页面已移除”）。

- 引导返回：设置“返回首页”按钮，点击跳转至官网首页；按钮样式突出，位置居中。

- 视觉优化：可选添加简洁的404插图（静态图片），避免页面过于单调。

## （四）非功能需求

1. **易用性**：界面导航清晰，用户可在3次点击内找到所需核心信息；操作逻辑简单，无学习成本，新生及非技术用户可轻松上手。

2. **安全性**：无敏感信息泄露，所有外部链接均为可信来源（如官方学习平台、正规竞赛官网）；表单提交功能仅收集必要信息，保护用户隐私。

3. **稳定性**：页面无卡顿、崩溃现象，图片、视频等资源加载稳定；不同浏览器（Chrome、Edge、Firefox、Safari）下均能正常显示与交互。

4. **可扩展性**：代码架构预留扩展空间，后续可新增功能模块（如成员博客、动态新闻、在线投票），无需大幅重构代码。

5. **美观性**：界面简洁大方，科技感强；色彩搭配协调，避免高饱和度、刺眼色彩；字体层级清晰，正文、标题、注释区分明确，阅读体验良好。

# 三、界面规划

## （一）整体界面架构

所有页面采用统一的“顶部导航栏 + 页面主体 + 底部页脚”三段式架构，保证风格一致性与用户体验连贯性。

- 顶部导航栏：固定定位，滚动时保持显示；背景色为白色或社团主题色，文字为深色（确保可读性）；Logo 位于左侧，导航入口居中，移动端折叠为汉堡菜单（右侧）。

- 页面主体：采用居中布局，最大宽度 1200px；内部模块通过栅格系统划分，间距统一（如上下模块间距 60px，左右间距 20px）；背景色为浅灰色或白色，提升阅读舒适度。

- 底部页脚：背景色为深色（如深灰、科技蓝），文字为白色；包含社团名称、版权信息（如“© 2024 XX大学计算机科技小组 版权所有”）、核心页面快速链接、备案信息（可选，如ICP备案号）。

## （四）交互设计规划
